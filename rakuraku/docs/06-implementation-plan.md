# 実装計画

AIエージェントがこのシステムを実装する順序を定義する。**このフェーズ順で進めること。** 各フェーズは前のフェーズの完成を前提とする。

各フェーズの終わりには「完了の確認」がある。それを満たしてから次に進む。

---

## フェーズ0: プロジェクトの土台

### やること
1. Next.js 15(App Router)+ TypeScript のプロジェクトを作成。
2. Tailwind CSS を導入し、`mockups/` の `:root` CSS変数を Tailwind の theme に移植する(`docs/03-screens.md` のデザイントークン参照)。
3. shadcn/ui を導入。
4. Supabase プロジェクトを用意し、`supabase/schema.sql` を適用してテーブルを作る。
5. Supabase クライアントを `lib/supabase/` に作る。
6. 環境変数(Supabase URL・キー)を設定。

### 完了の確認
- 空のホームページが表示される。
- Supabase に16テーブルができている。
- Tailwind でメインカラー `#1d4ed8` が使える。

---

## フェーズ1: 共通部品とホーム

### やること
1. 共通コンポーネントを作る。
   - `<TopHeader />` — ロゴ「楽」+ システム名 + ユーザー表示
   - `<PageBar title backTo />` — 「ホームに戻る」ボタン + 画面タイトル
   - `<BigButton />` — ホームの大きなカードボタン
   - `<Badge />`、`<RankBadge />` — 各種バッジ
2. ホーム画面 `/` を実装(`mockups/home.html` を忠実に再現)。10ボタン、5列2行。

### 完了の確認
- ホーム画面が `mockups/home.html` と同じ見た目になっている。
- 10個のボタンから各URLへ遷移できる(遷移先は空ページでよい)。

---

## フェーズ2: マスター(お客様・商品)

受発注はマスターに依存するので、先に作る。

### やること
1. **お客様マスター**
   - `/customers` 一覧(検索、絞り込みチップ、ページネーション)
   - `/customers/new`、`/customers/:code/edit` — 3ステップウィザード
   - Zod スキーマでバリデーション
   - お客様コードの自動採番
2. **商品マスター**
   - `/products` 一覧
   - `/products/new`、`/products/:code/edit`
   - 商品コードの自動採番

### 完了の確認
- お客様・商品を登録・編集・一覧表示できる。
- お客様のステップ3で帳票タイプ・締め日・税計算が保存される。
- モックアップと見た目が一致している。

---

## フェーズ3: 受注

### やること
1. `/orders/new` 「注文を受ける」
   - お客様検索・選択
   - 明細行(オートコンプリート、数量入力、自動計算)
   - 物理的な操作感(ドラッグ並び替え、Undo付き削除、↓キーで行追加)を `mockups/order-new.html` のとおり実装
   - 確定処理(`docs/05-workflows.md` フロー1)
2. `/orders` 注文一覧
3. `/orders/:id` 注文詳細
4. 採番ロジック `lib/utils/numbering.ts`

### 完了の確認
- 注文を受けると `sales_order` と `sales_order_line` ができる。
- 在庫商品は引当され、`order_at_sale` 商品は発注が自動作成される。
- 注文一覧・詳細で確認できる。

---

## フェーズ4: 発注・入荷・在庫

### やること
1. `/purchase-orders` 発注一覧(入荷状況のプログレスバー)
2. `/receivings` 「入荷を登録する」
   - 発注書を選び、入荷数量・ロット・賞味期限を入力
   - 確定処理(`docs/05-workflows.md` フロー2)
3. `/stocks` 「在庫を見る」
   - 商品ごと/ロットごとの表示切替
   - 期限警告(60日・14日)

### 完了の確認
- 入荷を登録すると在庫が増える。
- 在庫画面で実在庫・引当中・引当可能・期限が見える。
- 発注の入荷状況が更新される。

---

## フェーズ5: 納品・売上

### やること
1. `/deliveries` 「納品する」
   - 納品対象の受注を複数選択して一括納品
   - 確定処理(`docs/05-workflows.md` フロー3)
2. `/sales` 「売上を見る」
   - 期間サマリー、納品書一覧

### 完了の確認
- 納品を確定すると在庫が実減し、`sales_invoice` ができる。
- 受注のステータスが納品済み/一部納品に変わる。
- 売上画面で納品実績が見える。

## フェーズ6: 請求

### やること
1. `/billing` 「請求の業務」
   - ステップ1: 締め期間の指定と集計
   - ステップ2: お客様ごとの請求内容一覧
   - ステップ3: 請求書の一括発行
   - 確定処理(`docs/05-workflows.md` フロー4)

### 完了の確認
- 月次の締め集計ができる。
- `billing_statement` と `billing_statement_line` ができる。
- お客様の帳票タイプに応じて発行形式が切り替わる。

## フェーズ7: 帳票PDF

各画面ができてから、帳票出力を実装する。

### やること
1. `@react-pdf/renderer` を導入。
2. `lib/documents/` に5種類の帳票テンプレートを作る。
   - レイアウトの寸法・項目位置は `documents/` のPDF5種と `document-templates/make_documents.py` を基準にする。
3. 帳票生成のRoute Handler を `app/api/documents/` に作る。
4. 各画面に出力ボタンを足す。
   - 発注詳細 → 「発注書を印刷」
   - 注文詳細 → 「受注伝票を印刷」
   - 請求業務 → 「請求書を発行」「請求一覧表を印刷」

### 完了の確認
- 5種類の帳票PDFが、`documents/` のサンプルと同じレイアウトで出力される。
- 発行元情報(会社名・登録番号・振込先)が設定から差し込まれる。
