# プロジェクト背景

## このプロジェクトが生まれた理由

株式会社プロスパは、日用品・雑貨・食品などを扱うトン屋(問屋)である。小売店や薬局などの得意先から注文を受け、メーカーや一次卸から商品を仕入れ、得意先へ納品する。この受発注の業務を、長年、現行のシステムで回してきた。

現行システムには次の課題があった。

- 操作が複雑で、新しく入った事務員が一人で使えるようになるまで時間がかかる。
- 画面の言葉が専門的で、何をする画面なのか直感的にわからない。
- 帳票(発注書・請求書など)は出力できるが、システム全体の操作性が現代の水準から見て古い。

そこで、現行システムの**業務の流れと帳票はそのまま引き継ぎながら**、操作性を一から作り直した後継システムを新規開発することにした。それが「らくらく受発注システム」である。

## このシステムの最重要目標

**ITリテラシーが低い人が、マニュアルなしで使えること。**

これは「できれば達成したい」目標ではなく、このプロジェクトの存在理由そのものである。現場の事務員には、PC操作が得意でない人、年配の人も多い。その人たちが、研修や分厚いマニュアルなしに、画面を見ただけで「次に何をすればいいか」がわかる——それを実現できなければ、このシステムを作る意味がない。

技術的な洗練さ、機能の豊富さ、画面の格好良さは、すべてこの目標の下位にある。判断に迷ったら「リテラシーの低い人が迷わないのはどちらか」で決める。

## 誰が使うのか

| 利用者 | 役割 | システムでやること |
|--------|------|---------------------|
| 受発注の事務員 | 注文の受付、発注、納品処理 | 注文を受ける、発注、入荷、納品 |
| 経理担当 | 請求・入金管理 | 請求業務、入金消込 |
| 管理者 | マスター管理、全体把握 | お客様・商品マスター、売上確認 |

主な利用環境は**事務所のPC(マウス操作)**。タブレットやスマホは当面の主対象ではないが、画面はレスポンシブに作る。

## 何を引き継ぎ、何を変えるか

### 引き継ぐもの(現行システムと同じ)
- 業務の流れ(注文→発注→入荷→納品→請求→入金)
- 帳票5種類のレイアウト(発注書・受注伝票・請求書・請求書+納品書・請求一覧表)
- 採番の考え方、税率の扱い(10%・8%)

### 新しくするもの
- 操作性・画面デザインのすべて。業務動詞のラベル、ホームを中心にした導線、3ステップウィザードなど。
- 技術基盤(Next.js + Supabase のモダンな構成)。

## このシステムが扱う業務の範囲

**範囲に含む**:受発注、在庫管理(引当・入荷・実減)、納品、売上計上、月次請求、入金消込、お客様・商品マスター、帳票出力。

**範囲に含まない(当面)**:会計仕訳との連携、給与・人事、EC連携、複数倉庫の本格運用、お客様向けのオンライン発注画面。

## 開発の進め方

この後継システムは、AIエージェント(Claude Code)によって開発される。そのため、このドキュメント群は人間の開発者だけでなく、AIエージェントが読んで実装できる粒度で書かれている。仕様の曖昧さを減らし、判断の根拠を明示することを重視している。
