# 画面仕様書

全 13 画面の仕様。実装時は必ず `mockups/` 配下の HTML を参照し、レイアウト・文言・色を踏襲すること。

## 共通仕様

### 共通レイアウト
すべての画面に以下の構造が含まれる(ホーム画面以外)。

```
┌─────────────────────────────────────────────┐
│ [楽] らくらく受発注システム      [山田太郎] │  ← top-header
├─────────────────────────────────────────────┤
│ [← ホームに戻る]   画面タイトル              │  ← page-bar
├─────────────────────────────────────────────┤
│                                              │
│           メインコンテンツ                   │  ← content
│                                              │
└─────────────────────────────────────────────┘
```

### 共通コンポーネント
- `<TopHeader />`:ロゴ + システム名 + ユーザー情報
- `<PageBar title="..." backTo="..." />`:戻るボタンとタイトル
- `<BigButton variant="primary|default" />`:大きな立体ボタン
- `<Chip active />`:絞り込みチップ
- `<Badge variant="success|warning|danger|info|muted" />`:状態バッジ
- `<RankBadge rank="A" />`:ランクバッジ
- `<RowAction />`:テーブル行内のアクションボタン
- `<Pagination />`

### デザイントークン
すべてのスタイルはモックアップの `:root` で定義された CSS 変数を Tailwind の theme に移植する。

```
--primary: #1d4ed8
--primary-hover: #1e40af
--primary-light: #dbeafe
--primary-lighter: #eff6ff
--bg-base: #f1f5f9
--bg-surface: #ffffff
--bg-muted: #f1f5f9
--border-light: #e2e8f0
--border-default: #cbd5e1
--text-primary: #0f172a
--text-secondary: #475569
--text-muted: #64748b
--success: #059669 / --success-light: #d1fae5
--warning: #d97706 / --warning-light: #fef3c7
--danger: #dc2626 / --danger-light: #fee2e2
--info: #0284c7 / --info-light: #e0f2fe
```

## 1. ホーム (`/`)

**ファイル:** `mockups/home.html`

### 目的
ユーザーが業務作業を選ぶ起点。10 個の大きなボタンで全機能を表現する。

### レイアウト
5 列 × 2 行のカードグリッド。狭い画面では 3 列 → 2 列 → 1 列に折り返す。

```
上段(日々の作業+確認)    : 注文を受ける | 納品する   | 入荷を登録する | 注文を見る | 在庫を見る
下段(マスター・実績・請求): 発注を見る   | お客様を管理 | 商品を管理     | 売上を見る | 請求の業務
```

### 各ボタンの遷移先

| ラベル | 遷移先 | 色 |
|--------|--------|-----|
| 注文を受ける | `/orders/new` | blue |
| 納品する | `/deliveries` | green |
| 入荷を登録する | `/receivings` | amber |
| 注文を見る | `/orders` | orange |
| 在庫を見る | `/stocks` | indigo |
| 発注を見る | `/purchase-orders` | rose |
| お客様を管理する | `/customers` | purple |
| 商品を管理する | `/products` | teal |
| 売上を見る | `/sales` | pink |
| 請求の業務 | `/billing` | violet |

### データ要件
なし(静的)。将来は警告サマリー(期限切れ件数、納品待ち件数など)を表示する余地あり。

## 3. お客様の登録/編集 (`/customers/new`, `/customers/:code/edit`)

**ファイル:** `mockups/customer-edit.html`

### 目的
お客様情報の登録と編集。3 ステップのウィザード形式。

### ステップ構成
- **ステップ 1: 基本情報**(名前、フリガナ、コード、ランク、担当者、状態)
- **ステップ 2: 連絡先**(郵便番号、住所、建物名、電話、FAX、先方担当者、メール、届け先メモ)
- **ステップ 3: 請求の設定**(請求書の種類、締め日、支払い予定、消費税の計算方法、端数処理)

### 主要操作
- 各ステップで「次へ進む」「前に戻る」ボタンでステップ間を移動
- ステップ表示の丸は、現在=青、完了=緑チェックに変わる
- 「キャンセル」で `/customers` に戻る(変更破棄、確認ダイアログ)
- ステップ3の「この内容でお客様を登録する」で確定

### バリデーション
- 必須項目:名前、ランク、担当者、状態(ステップ1)、住所、電話(ステップ2)、請求書の種類、締め日、消費税の計算方法、端数処理(ステップ3)
- フリガナはカタカナのみ
- お客様コードは空欄なら自動採番、入力する場合は数字 6 桁で重複不可

### ランク選択 UI
4 つの大きなカードボタン(A: 特に大切 / B: 大切 / C: 標準 / D: 小規模)。

### 状態選択 UI
ラジオボタン形式(取引中 / 休止中 / 取引なし)。

### 請求書の種類の選択 UI(ステップ3)
2 つの大きな選択カード。それぞれ紙面のミニプレビュー付き。
- **請求書のみ**(`invoice_only`):請求書を1枚だけ発行
- **請求書＋納品書**(`invoice_delivery`):1枚に請求書と納品書を上下2段組で印刷

ここで選んだ値が `customer.invoice_format` に保存され、請求業務画面での帳票発行時に形式が自動で切り替わる。

### 締め日・税計算の設定(ステップ3)
- 締め日:プルダウン(末日締め / 20日締め / 25日締め / 15日締め / 10日締め / 5日締め)。`customer.closing_day` に保存
- お支払い予定:プルダウン(翌月末 / 翌月20日 / 翌月10日 / 翌々月末)。`customer.payment_cycle` に保存
- 消費税の計算方法:ラジオ(明細ごと / 伝票ごと / 請求書ごと)。`customer.invoice_tax_type` に保存
- 消費税の端数:ラジオ(切り捨て / 四捨五入 / 切り上げ)。`customer.tax_rounding` に保存

## 7. 注文を受ける (`/orders/new`)

**ファイル:** `mockups/order-new.html`

### 目的
新規受注の登録。本システムで最も使用頻度が高い画面。

### セクション構成
1. **どのお客様からの注文ですか?**:お客様の検索・選択
2. **注文の日付と納品日**:注文日、納品予定日、届け先
3. **注文する商品を選んでください**:明細行(複数行)
4. **メモ(任意)**:伝票メモ

### 明細行の動作
- 商品名/商品コード/JAN コードで検索しオートコンプリート
- 商品選択時に売上単価・税率を自動セット(明細で上書き可)
- 数量を入力すると金額を自動計算
- 行ごとの削除ボタン
- 「商品の行を追加する」で新規行を追加

### 自動計算
- 各行:`金額 = 単価 × 数量`
- フッター:税率ごとの小計、消費税、合計を表示

### 在庫チェック
- 商品行ごとに「引当可能数」をリアルタイム表示
- 数量が引当可能数を超えても警告のみ(受注は通せる)

### 確定処理(「この内容で注文を確定する」押下時)
1. `sales_order` を作成(status = pending)
2. `sales_order_line` を作成(明細スナップショット)
3. 在庫商品の場合:FIFO で `sales_order_line_allocation` を作成、`product_stock.quantity_allocated` を増やす、`stock_movement` に `allocate` を記録
4. `default_order_type = order_at_sale` の商品は自動で `purchase_order` を作成

## 8. 納品する (`/deliveries`)

**ファイル:** `mockups/delivery.html`

### 目的
納品予定の受注を一覧から選び、まとめて納品確定する。

### 主要操作
- 期間フィルタ(今日まで / 今週まで / すべて)
- 検索(伝票番号・お客様名)
- 各注文行のチェックボックスで選択
- 下部の固定バーに「納品を確定する」ボタン

### 表示要素
- 注文番号、お客様、納品予定日、商品品目数、金額
- 緊急度バッジ(本日が納品日 → 赤、明日が納品日 → 黄)

### 確定処理(複数選択して納品確定時)
選択された各受注について:
1. `sales_invoice` を新規作成(納品書番号を自動採番)
2. `sales_invoice_line` を `sales_order_line` から複製
3. `product_stock.quantity_on_hand` と `quantity_allocated` を両方減らす
4. `stock_movement` に `out` と `deallocate` を記録
5. `sales_order.status` を更新(全納品なら fulfilled、一部なら partial)

### 部分納品(将来対応)
現在は受注全体を一括納品する想定。将来は明細単位の部分納品に対応する余地を残す。

## 12. 入荷を登録する (`/receivings`, `/receivings?po=:id`)

**ファイル:** `mockups/receivings.html`

### 目的
仕入先からの納品書を見ながら、発注に対する入荷を確定する。ロット商品はロット番号と賞味期限も入力。

### 上部:対象の発注書情報
- 発注番号、仕入先、発注日、入荷予定日、合計金額、これまでの入荷率
- 「別の発注書を選ぶ」ボタンで切り替え

### 入荷明細の入力
発注の各明細行に対して、以下を入力:
- **入荷数量**(必須)
- **ロット番号**(空欄で自動採番、ロット管理商品のみ表示)
- **賞味期限**(必須、ロット管理商品のみ表示)
- **入荷日**(既定: 今日)
- **メモ**(任意)

ロット管理なしの商品は「期限管理」マークが付かず、ロット番号と賞味期限の入力欄が表示されない。

### 確定処理(「この内容で入荷を確定する」押下時)
入荷数量が 0 でない明細について:
1. `product_stock` を検索(同じ仕入先・商品・期限の組み合わせがあれば既存行に加算、なければ新規作成)
2. `quantity_on_hand` を増やす
3. `stock_movement` に `in` を記録
4. `purchase_order_line.received_quantity` を更新
5. `purchase_order.status` を更新(全量入荷なら `received`、一部なら `partial`)

### 分納対応
入荷数量を発注数量より少なく入力すれば、残りの入荷を後日同じ発注書から続けられる。

## 13. 請求の業務 (`/billing`)

**ファイル:** `mockups/billing.html`

### 目的
月次の締め処理を行い、請求書・請求一覧表を発行する。経理担当が月に一度使う画面。

### 画面構成
処理の流れに沿った 3 ステップを縦に並べる。上から順に進める。

### ステップ1: 締める期間を決める
- 締め日のプルダウン(毎月20日締め など)
- 対象期間の開始日・終了日(締め日)
- 「この期間で集計する」ボタン
- 集計後、サマリーを表示:対象のお客様数、売上伝票の枚数、今回の請求合計

### ステップ2: 請求の内容を確認する
- 締め対象のお客様ごとに 1 行の一覧
- 表示カラム:コード、お客様、前回請求、入金、繰越、今回売上、消費税、今回請求額、回収予定
- この一覧がそのまま「請求一覧表」(帳票5)の元データになる
- 「請求一覧表を印刷する」ボタン → 帳票5のPDFを出力

### ステップ3: 請求書を発行する
- 発行対象のお客様をチェックボックスで選択(「すべて選ぶ」あり)
- 各行に帳票タイプのバッジを表示:
  - 青「請求書のみ」(`customer.invoice_format = invoice_only`)
  - 紫「請求書+納品書」(`customer.invoice_format = invoice_delivery`)
- 帳票タイプはお客様マスターの設定で自動判定。ユーザーは選ばない
- 下部の発行バーに「発行前にプレビュー」「請求書をまとめて発行する」

### 確定処理(「請求書をまとめて発行する」押下時)
選択された各お客様について:
1. 対象期間の `sales_invoice`(納品書)を集計
2. `billing_statement` を作成(請求書番号を自動採番、前回請求・入金・繰越を計算)
3. `billing_statement_line` を税率別に作成
4. `customer.invoice_format` に従って、請求書のみ/請求書+納品書 のPDFを生成
5. `billing_statement.status` を `issued`(請求済み)に更新

### データ要件
- `billing_statement`、`billing_statement_line`
- 集計元として `sales_invoice`、`sales_invoice_line`
- `customer`(締め日・帳票タイプ・税計算区分)
- `payment`、`payment_allocation`(前回請求・入金・繰越の計算)
