# 業務概要と用語集

## 業務の全体像

トン屋(問屋)業務における受発注管理。お客様(得意先)から注文を受け、必要に応じて仕入先に発注し、商品を納品して売上を計上する。月末に請求書を発行し、入金を消し込む。

```
┌──────────┐  注文  ┌────────┐         ┌──────────┐
│ お客様   │──────→│ 当社   │←─入荷──│ 仕入先   │
│ (得意先) │←─納品─│ (トン屋)│──発注─→│          │
└──────────┘       └────────┘         └──────────┘
                       │
                       ▼
                  在庫(倉庫)
```

## 業務フロー(基本)

1. **注文を受ける**:お客様から注文を受け、受注を登録する
2. **在庫の引当**:在庫がある商品は受注時に引当(仮押さえ)、ない商品は仕入先に発注
3. **入荷**:仕入先から商品が届いたら、発注に対して入荷を確定し、在庫を加算
4. **納品**:お客様の納品予定日に商品を出荷し、納品を確定する。これにより:
   - 引当数が減り、実在庫数も減る
   - 売上(納品書)が自動生成される
5. **請求**:月次で締め日に請求書を発行する
6. **入金**:お客様から入金があったら、請求と消し込む

## 重要な業務ルール

### 在庫の動き
- **受注時**:`product_stock.quantity_allocated` を増やす(`stock_movement` に `allocate` タイプで記録)
- **納品時**:`product_stock.quantity_on_hand` と `quantity_allocated` を両方減らす(`out` と `deallocate` の 2 つの movement)
- **入荷時**:`quantity_on_hand` を増やす(`in` タイプ)
- **受注キャンセル**:`quantity_allocated` を減らす(`deallocate` タイプ)
- **棚卸補正・廃棄**:将来対応(`adjust`、`dispose` タイプを予約)

### FIFO の引当
受注時に複数のロット(`product_stock` 行)から自動で引き当てる。

優先順位:
1. 賞味期限が近いロットを優先
2. 期限が同じなら入荷日が古いロットを優先
3. 期限なし商品はロットを区別せず 1 行にまとめる

例:商品 A を 30 個受注。期限 2026/05/10 のロットが 20 個、期限 2026/06/15 のロットが 50 個ある場合、まず 5/10 ロットから 20 個引き当て、残り 10 個を 6/15 ロットから引き当てる。1 つの受注明細に対して `sales_order_line_allocation` が 2 行作られる。

### 在庫不足の扱い
- 受注時に引当可能数を超える注文を受けても、警告は出すが受注は通す
- `quantity_allocated` がマイナスの「引当可能数」を許容(オーバー引当)
- ただし納品時には実在庫が必要なので、その時点で在庫がなければ納品確定できない

### 価格の決定ロジック
明細行の単価は以下の順で決まる(現時点)。

1. 明細で手動入力された値があればそれを使う
2. なければ商品マスターの `default_sales_unit_price`
3. それでも未設定なら 0(警告表示)

将来追加予定:お客様別単価、ランク別掛け率、特売価格。

### 税額計算
- 税率は 10% と 8% のみ(明細行で個別に指定可)
- 端数処理:1 伝票単位で集計後、税率ごとに小計を出し、それぞれ四捨五入
- 商品マスターに `default_tax_rate` を持ち、明細で上書き可能

### 期限警告
- 期限まで 60 日以下:黄色警告
- 期限まで 14 日以下:赤警告
- 商品マスターで個別に閾値を変更可能

### 受注番号・伝票番号の採番
- 受注番号:`8` + 連番 8 桁(例: `800028791`)
- 発注番号:`P` + 連番 8 桁(例: `P00001284`)
- 納品書番号:`N` + 連番 8 桁(例: `N00012841`)
- 請求書番号:`B` + YYYYMM + 連番(例: `B20260400123`)

## ステータス遷移

### 受注のステータス
```
[作成中] → [納品待ち] → [一部納品] → [納品済み]
                ↓
            [取消し済み]
```

### 発注のステータス
```
[作成中] → [発注済み] → [一部入荷] → [全量入荷]
                ↓                      ↑
            [取消し済み]           (入荷遅延の場合は赤表示)
```

### 請求のステータス
```
[未請求] → [請求済み] → [一部入金] → [入金済み]
```

## 採番・コード体系

- お客様コード:数字 6 桁(例: `000199`)、空欄なら自動採番
- 商品コード:数字 8 桁(例: `25253053`)、空欄なら自動採番
- 仕入先コード:数字 6 桁
- ロット番号:`YYYYMMDD-連番3桁`(例: `20260424-001`)、空欄なら自動採番
