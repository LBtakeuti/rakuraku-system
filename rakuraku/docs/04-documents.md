# 帳票仕様書

現行システム(株式会社プロスパ)の実物伝票5種類をもとに作成。実装時はこの仕様に従い、現行レイアウトを忠実に踏襲すること。出力形式はすべて PDF(画面から生成、印刷・ダウンロード)。

## 帳票一覧

| 帳票 | 用途 | 出力元データ | 出力する画面 |
|------|------|-------------|-------------|
| 発注書 | 仕入先へ送る注文書 | purchase_order | 発注を見る → 詳細 |
| 受注伝票 | 社内向け受注控え | sales_order | 注文を見る → 詳細 |
| 請求書(単体) | お客様への請求 | billing_statement | 請求業務(月次) |
| 請求書+納品書 | 請求書と納品書を1枚に2段組 | billing_statement + sales_invoice | 請求業務(月次) |
| 請求一覧表 | 全得意先の請求状況(社内資料) | billing_statement 月次集計 | 請求業務(月次) |

## 共通の発行元情報

すべての帳票に印字される自社情報(現行実物より):

```
株式会社プロスパ
登録番号 T6011801020915
121-0831 東京都足立区舎人5-16-10
TEL: 03-5856-8263   FAX: 03-5856-8273
【お振込先】三菱UFJ銀行/千住支店 普通預金 4814091
```

これらは設定マスター(`company_setting` など)に持たせ、帳票生成時に差し込む。ハードコードしないこと。

---

## 1. 発注書

**用途:** 仕入先に送る注文書。「必ず出荷日・納品日のご返信をお願いいたします」と返信依頼が入る。

### ヘッダ部
- タイトル「発注書」(中央、囲み枠)
- 伝票番号(右上、例: 600026741)
- Page(例: 1/1)
- 伝票日付(例: 2026年 3月31日)
- **※納期**(例: 2026年 4月 3日)+ 時間指定(例: AM必着)— 大きめ強調表示
- 仕入先の営業所名(左上、例: 北関東営業所)
- 仕入先の住所・電話(左、薄字)
- 自社情報(右、発行元)

### 明細テーブル
| 列 | 内容 |
|----|------|
| No. | 行番号(例: 001) |
| 商品名/JANコード | 商品名(上段)とJANコード(下段)の2段 |
| 入数 | 1ケースの入数(例: 48) |
| ケース | ケース数(例: 100) |
| 発注数量 | 入数×ケース(例: 4,800) |
| 備考 | 自由記入 |

### フッタ部
- 送り先(届け先名、例: 平塚配送センター ㈱アエナ)
- 住所、電話
- 送り主(自社名)、送り主住所、送り主電話番号
- 返信欄:「__月__日 出荷」「__月__日 着」(仕入先が記入して返送する欄)
- 定型文「いつもお世話になっております。必ず出荷日・納品日のご返信をお願いいたします。」

### データ対応
- 伝票番号 → `purchase_order.purchase_order_no`
- 伝票日付 → `purchase_order.order_date`
- 納期 → `purchase_order.expected_delivery_date`(+ 時間指定欄を新規追加)
- 仕入先 → `supplier`
- 明細 → `purchase_order_line`(入数は `product.units_per_case` から)
- 送り先 → 受注に紐づく `delivery_address`

### 追加が必要なフィールド
- `purchase_order.delivery_time_note`(納期の時間指定:「AM必着」など)

## 2. 受注伝票

**用途:** 社内で受注内容を確認するための控え。

### ヘッダ部
- タイトル「受注伝票」(中央、囲み枠)
- 伝票番号(右上、例: 600026741)
- Page(例: 1/1)
- 伝票日付(例: 2026年 3月31日)
- お客様コード(左上、例: 000598)
- お客様名(例: 株式会社アエナ 様)
- 送り先・住所・電話(届け先)
- 送り主(自社名)、送り主住所・電話

### 明細テーブル
| 列 | 内容 |
|----|------|
| No. | 行番号 |
| 商品名 | 商品コード + JANコード(上段)、商品名(下段) |
| 入数/ケース | 入数(上段)、ケース数(下段) |
| 発注数量 | 数量 |
| 単価 | 単価 |
| 売上金額 | 金額 |
| 発注番号/備考 | 紐づく発注番号(例: 600026695)+ 備考 |

### フッタ部
- 合計
- 消費税額
- 総合計
- 摘要欄

### データ対応
- 伝票番号 → `sales_order.order_no`
- お客様コード・名 → `customer`
- 明細 → `sales_order_line`
- 発注番号 → 紐づく `purchase_order.purchase_order_no`
- 合計・消費税・総合計 → `sales_order` の金額フィールド

## 3. 請求書(単体)

**用途:** お客様への月次請求書。お客様コードで管理。

### ヘッダ部
- タイトル「請求書」(中央)
- Page(例: 1/1)
- 締日(例: 2026年 3月31日 締)
- お支払期限(例: 2026年 4月30日)
- お客様の郵便番号・住所・社名(左上)
- お客様コード(例: 000072)
- 自社情報・登録番号・振込先(右)
- 定型文「毎度ありがとうございます。下記の通りご請求申し上げます。」
- 注記「インボイス制度開始に伴い各種手数料は貴社ご負担にてお願い致します。」

### 集計欄(ヘッダ下の横帯)
| 項目 | 内容 |
|------|------|
| 前回御請求額 | 前月の請求残 |
| 御入金額 | 入金された額 |
| 繰越金額 | 繰り越された額 |
| 税率 | 10% と 8% の2行 |
| 御買上額 | 税抜の買上額(税率別) |
| 消費税額 | 税率別の消費税 |
| 御買上計 | 税込合計 |
| 今回御請求額 | 最終的な請求額(例: ¥12,672) |

### 明細テーブル
| 列 | 内容 |
|----|------|
| 伝票日付 | 納品日(例: 2026/03/05) |
| 伝票No. | 納品書番号(例: 100032962) |
| 品番・品名 | 商品コード+商品名。下段に納品先メモ(例: セイセイ薬品様納品分) |
| 数量 | 数量 |
| 単価 | 単価 |
| 御買上額 | 金額 |

### データ対応
- 締日・支払期限 → `billing_statement.period_to`, `due_date`
- お客様コード → `customer.customer_code`
- 前回御請求額・御入金額・繰越金額 → `billing_statement` の対応フィールド
- 税率別の集計 → `billing_statement_line` を税率別に集計
- 明細(伝票日付・伝票No.) → 紐づく `sales_invoice`

## 4. 請求書+納品書(2段組)

**用途:** 1枚の用紙に「請求書」(上段)と「納品書」(下段)を印刷する形式。お客様によってこの形式を使う。

### レイアウト
A4縦1枚を上下に分割。上段が請求書、下段が納品書。両方とも同じ伝票番号(例: 100033114)。

### 上段:請求書
- タイトル「請求書」、日付、伝票番号、Page
- お客様の住所・社名、お客様コード
- 自社情報・振込先・登録番号
- 検印欄(右上、空欄の枠)
- 明細テーブル:品番・品名 / 数量(入数・箱数も) / 単価 / 金額 / ※ / 備考
- 摘要欄
- 税率別集計:「10% 税抜合計 / 消費税額」「8% 税抜合計 / 消費税額」「総合計金額」

### 下段:納品書
- タイトル「納品書」、構成は請求書とほぼ同じ
- 定型文「毎度ありがとうございます。下記の通り納品致しましたのでご査収下さい。」
- 明細・集計欄は上段と同一内容

### 明細テーブルの詳細
| 列 | 内容 |
|----|------|
| 品番・品名 | 品番(上段、JANコード)、品名(下段)。入数も表示 |
| 数量 | 箱数(上段)と総数量(下段) |
| 単価 | 単価 |
| 金額 | 金額 |
| ※ | 軽減税率マークなど |
| 備考 | 納品メモ(例: 3/16納品 NO.xxx、運送会社名、営業所入れ 等) |

### データ対応
- この形式を使うかどうかは `customer.invoice_format` で判定(下記参照)
- 請求書部分 → `billing_statement`
- 納品書部分 → `sales_invoice`
- 両者は同じ伝票番号で紐づく

## 5. 請求一覧表

**用途:** 全得意先の請求状況を一覧化した社内管理資料。月次の締め後に出力。

### ヘッダ部
- タイトル「請求一覧表(請求済)」
- 作成年月日
- 締日(例: 2026年 3月20日 締)
- ページ番号
- 売上伝票枚数・回収枚数の集計(総合計行)

### 明細テーブル(得意先ごとに1行)
| 列 | 内容 |
|----|------|
| 得意先コード | 例: 000113 |
| 得意先名 | 例: 株式会社○○商店 |
| 本社/外税 区分 | 税計算区分 |
| 売上伝票枚数 | |
| 回収予定日 | 例: 2026-4-20 |
| 前回請求 | |
| 入金 | |
| 入金時手数料 | |
| 繰越 | |
| 諸掛 | 諸掛(雑費) |
| 返品・値引 | |
| 売上 | |
| 請求値引 | |
| 消費税 | |
| 御買上金額 | |
| 今回請求 | |

### 集計
- 最上部に総合計行(全得意先の合計)

### データ対応
- 月次の `billing_statement` を全件集計
- 得意先ごとに前回請求・入金・繰越・売上・今回請求を並べる
- これは一覧画面でもあり、印刷帳票でもある

## お客様マスターへの追加項目

帳票の使い分けのため、`customer` テーブルに以下を追加する。

```
customer に追加:
  - invoice_format text  -- 請求書の形式
      'invoice_only'      = 請求書のみ(帳票3)
      'invoice_delivery'  = 請求書+納品書の2段組(帳票4)
  - invoice_tax_type text -- 税計算区分(本社/外税 など。請求一覧表の区分)
  - closing_day int       -- 締日(20日締、月末締 など。請求書の締日計算に使用)
```

`product.units_per_case`(入数)は発注書・受注伝票・納品書すべてで使うため必須。

## 帳票生成の技術方針

- PDF 生成は サーバーサイド(Next.js の Route Handler / Server Action)で行う
- ライブラリ候補:`@react-pdf/renderer`(React で帳票レイアウトを組める)または HTML→PDF(`puppeteer` 系)
- 現行レイアウトの忠実な再現が要件なので、罫線・項目位置を実物に合わせる
- A4縦が基本。請求書+納品書は1枚2段組
- 帳票テンプレートは画面とは別に `src/lib/documents/` 配下にまとめる
- フォントは明朝・ゴシックの混在(実物に合わせる)。日本語フォントの埋め込みが必要

## 出力ポイントのまとめ(画面への組み込み)

| 画面 | 追加するボタン |
|------|--------------|
| 発注を見る → 詳細 | 「発注書を印刷」 |
| 注文を見る → 詳細 | 「受注伝票を印刷」 |
| 請求業務(新規画面) | 「請求書を発行」「請求一覧表を印刷」 |
| 納品する → 完了後 | (請求書+納品書形式のお客様は請求業務側で出力) |

「請求業務」画面は未設計。月次の締め処理を行い、請求書・請求一覧表を出力する画面として新規に設計が必要。
